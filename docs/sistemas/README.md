# Sistemas Informáticos

En esta carpeta se recoge la parte del proyecto intermodular correspondiente al módulo de **Sistemas Informáticos**.

## Objetivo

El objetivo de esta parte del proyecto es describir el entorno técnico necesario para ejecutar la aplicación **Lead Intelligence Manager** de forma realista y coherente con su funcionamiento.

La aplicación se plantea como una **app de escritorio/local** desarrollada en **Java con Swing**, conectada mediante **JDBC** a una base de datos **MySQL** local.

## Contenido de la carpeta

### `informe-tecnico.md`
Documento principal del bloque de Sistemas. En él se explica:

- descripción general de la aplicación;
- tipo de sistema donde se ejecuta;
- requisitos de hardware;
- sistema operativo utilizado y recomendado;
- software necesario;
- instalación del entorno;
- ejecución de la aplicación;
- usuarios, permisos y estructura de carpetas;
- mantenimiento básico;
- protección mínima;
- evidencias de funcionamiento.

### `capturas/`
Carpeta destinada a guardar capturas que demuestren el funcionamiento real del entorno y de la aplicación.

## Relación con el proyecto

Esta parte está directamente conectada con el resto del intermodular, ya que permite justificar:

- en qué tipo de equipo se ejecuta la aplicación;
- qué programas son necesarios para ponerla en marcha;
- cómo se instala y configura el entorno;
- cómo se conecta la aplicación Java con MySQL;
- qué evidencias demuestran que el sistema funciona correctamente.

## Tipo de entorno definido

El proyecto se ejecuta en un **PC de usuario**, utilizando el mismo equipo para:

- desarrollo en IntelliJ IDEA;
- ejecución de MySQL Server;
- gestión de la base de datos mediante MySQL Workbench;
- ejecución de la aplicación Java.

Este planteamiento resulta simple, realista y adecuado al nivel del proyecto.
